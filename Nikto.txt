# Nikto (Web Server Scanner)

## Basic Scanning
nikto -h http://example.com
nikto -h https://example.com
nikto -h 192.168.1.1
nikto -h http://example.com:8080
nikto -h http://example.com -p 8080,8443
nikto -h http://example.com -p 80,443,8080,8443

## Input from File
nikto -h targets.txt
nikto -h targets.txt -o results.txt
nikto -h targets.txt -Format txt

## SSL and HTTPS
nikto -h https://example.com -ssl
nikto -h example.com -p 443 -ssl
nikto -h example.com -ssl -nossl
nikto -h https://example.com -ssl -Tuning x

nikto -h http://example.com -id admin:password123

## Common Flags
- `-h` - Target host; accepts IP, hostname, or URL
- `-p` - Target port(s); accepts single ports or comma-separated list
- `-ssl` - Force SSL/HTTPS mode on the connection
- `-id` - HTTP Basic Authentication credentials in `user:password` format
- `-C` - Send custom cookies with every request
- `-useproxy` - Route all traffic through the specified HTTP or SOCKS proxy
- `-Tuning` - Control which test categories to run; values 0-9 and x select test types (1=Interesting Files, 2=Misconfiguration, 3=Info Disclosure, 4=Injection, 5=Remote File Retrieval, 6=Denial of Service, 7=Remote Shell, 8=Command Execution, 9=SQL Injection, 0=File Upload, x=Reverse Tuning/Exclude)
- `-evasion` - IDS evasion technique (1=Random URI encoding, 2=Directory self-reference, 3=Premature URL ending, 4=Prepend long random string, 5=Fake parameter, 6=TAB as request spacer, 7=Change the case of the URL, 8=Use Windows path separator)
- `-Plugins` - Specify individual plugins to run by name; use `-list-plugins` to see all
- `-list-plugins` - Display all available plugins with descriptions
- `-vhost` - Set a virtual hostname in the Host header for IP-based scanning
- `-useragent` - Set a custom User-Agent string for all requests
- `-o` - Output file path to save scan results
- `-Format` - Output format: txt, xml, htm, csv, nbe, json
- `-Display` - Control verbosity of output (V=verbose, 1=redirects, 2=cookies, E=errors, P=HTTP progress)
- `-timeout` - Per-request connection timeout in seconds (default: 10)
- `-Pause` - Delay in seconds between each request to reduce detection
- `-maxtime` - Maximum total time in seconds to run the scan
- `-nossl` - Disable SSL even on ports that typically use it
- `-no404` - Disable 404 detection to ignore custom error pages
- `-update` - Update the plugins and databases from the CIRT.net server
